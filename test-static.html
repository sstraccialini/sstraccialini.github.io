Deployment Instructions

  1. Rename the static folder to docs
  2. Push the docs folder to your GitHub repository
  3. In your repository Settings → Pages, set the source to the main branch and /docs folder
  4. Wait for GitHub Pages to build your site (usually takes 1-2 minutes)

Common Issues

  • Paths in HTML files still reference /_next/ instead of ./next/
  • Missing .nojekyll file (GitHub Pages ignores folders starting with underscore)
  • Not correctly set up in GitHub Pages settings
