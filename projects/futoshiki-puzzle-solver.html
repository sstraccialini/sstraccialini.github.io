# Futoshiki Puzzle Solver using Evolutionary Programming ## Overview This project implements an **Evolutionary Programming** approach to solve **Futoshiki puzzles**, a Japanese logic-based combinatorial number puzzle. The solver uses genetic algorithms and advanced techniques like island models to tackle the complex constraint satisfaction problem that Futoshiki presents. The puzzle is played on an N × N square grid (commonly 4×4, 5×5, or 6×6) 2. Some cells contain predefined numbers that cannot be changed 3. Fill each cell with a number from 1 to N 4. Each number must appear exactly once in every row 5. Each number must appear exactly once in every column 6. Some cells are separated by inequality signs (< or >) that indicate relative ordering between adjacent cells 7. Any placed numbers must satisfy all given inequality constraints 8. The puzzle has a unique solution that respects all constraints ## Project Features ### Core Implementation - **FutoshikiPuzzle Class**: Represents and handles puzzle instances with constraints - **Individual Class**: Represents candidate solutions in the genetic algorithm - **Fitness Evaluation**: Counts constraint violations (row, column, and inequality) - **Genetic Operators**: Crossover and mutation for solution evolution ### Evolutionary Algorithms 1. **Standard Genetic Algorithm** - Population-based evolutionary approach - Elitism strategy (preserves top 10% of individuals) - Roulette wheel selection for parent selection - Crossover and mutation operators 2. **Island Model Genetic Algorithm** - Multiple isolated populations (islands) - Independent evolution with periodic migration - Ring topology for migration between islands - Enhanced diversity and escape from local optima ### Optimization Features - **Grid Search**: Systematic hyperparameter optimization - **Performance Analysis**: Fitness tracking and visualization - **Multiple Puzzle Sizes**: Support for 4×4, 5×5, and 6×6 grids ## Files Structure ``` ├── project.ipynb # Main Jupyter notebook with complete implementation ├── project_Straccialini.html # HTML export of the notebook └── README.md # This file ``` ## Key Components ### 1.
